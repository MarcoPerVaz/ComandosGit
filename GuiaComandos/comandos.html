# Comando Descripción Observaciones 
git init
 Crea la carpeta oculta .git donde se encuentre la ruta en consola y dónde se guarda toda la información de git, en pocas palabras inicializa el repositorio de Git 
git status
 Muestra el estado de los archivos:
Rojo: No agregados o modificados sin agregar
Verde: Agregados 
git log
 Muestra todos los commits de forma extendida creados hasta ahora ordenados del más reciente(arriba) al más antigüo (abajo) 
git diff
 Muestra los cambios que hubo en los archivos
Rojo: Archivo original
Verde: Cambios en el archivo original 
git reflog
 Muestra todo lo que se ha hecho en consola para regresar a un commit específico si así se desea, para usarlo se debe usar 
git reset --hard _hash_
git reset --hard 0808461
 
git rebase
 Reorganiza el repositorio
Une ramas, rehace commits, une commits, divide commits, cambia nombres de commits, borra commits, en pocas palabras rehace la historia de un proyecto
# Comando Descripción Observaciones 
git add .
 Agrega todos los archivos 
git add _nombreArchivo.extension_
 Agrega el archivo especificado Ejemplo: 
git add index.html
 
git add *._extension_
 Agrega todos los archivos del directorio actual con una extensión específica Ejemplo: 
git add *.png
 
git add "*._extension_"
 Agrega los archivos con una extensión específica de todo el proyecto/repositorio.
Lleva comillas dobles 		Ejemplo: 
git add "*.png"
 
git add _carpeta_/
 Agrega todos los archivos dentro de una carpeta específica Ejemplo: 
git add css/
Ejemplo: 
git add Curso/demo-02/css/
 
git add -A
 Agrega todos los archivos que fueron modificados Ejemplo: 
git add css/
Ejemplo: 
git add Curso/demo-02/css/
 
git add --all
 Agrega todos los archivos
Similar al git add . o git add -A 
git add _listaDeArchivos_
 Agrega la lista de archivos que especifiquemos Ejemplo: 
git add index.html style.css app.js
 
git add _carpeta_/*._extension_
 Agrega los archivos con extensión específica que estén dentro de una carpeta específica Ejemplo: 
git add pdfs/*.pdf
 
git add -u
 Actualizar el escenario (staged)
# Comando Descripción Observaciones 
git commit -m "nombreCommit"
 Crea un snapshot o captura del código Ejemplo: 
git commit -m "Primer commit"
 
git commit -am "nombreCommit"
 Agrega los archivos que Git ya esté pendiente de los cambios y crea el snapshot del código Ejemplo: 
git commit -am "Readme actualz"
 
git commit --amend -m "nombreCommit"
 Cambia el nombre del último commit Ejemplo: 
git commit --amend -m "Actualizamos el readme"
 
git commit
 Crea un snapshot pero permitiendo agregar un nombre de varias líneas Ejemplo: 
git commit
Dependiendo la configuración del editor de código, se muestra en la consola o en una pestaña aparte (en mi caso es una pestaña aparte de Vscode) 
git commit -a
 Crea un commit resolviendo conflictos al hacer merge

# Comando Descripción Observaciones 
git chekout -- .
 Reconstruye el proyecto a como estaba en el último commit
*Recupera código, archivos y carpetas borradas, excepto la carpeta .git (carpeta oculta, no eliminar a menos que desee dejar de usar Git o volver a crear el repositorio) 
git chekout -- _nombreArchivo.extension_
 Reconstruye el cambio del archivo especificado (las mayúsculas y minúsculas importan)
*Recupera el código a como estaba en el último commit 		Ejemplo: 
git checkout .. README.md
Ejemplo: 
git checkout .. Curso/demo-03/README.md
 
git checkout _nombreRama_
 Mueve el proyecto a la rama especificada Ejemplo: 
git checkout rama_villanos
 
git checkout -b _nombreRama_
 Crear una rama y moverse a esa rama con un solo comando Ejemplo: 
git checkout -b rama-villano
 
git checkout -b _hashRamaARecuperar_
 Recuperar una rama eliminada usando el hash o sha
*Al recuperar la rama, el nombre de la rama pasa a ser el Hash 		Ejemplo: 
git checkout -b 51b4202
# Comando Descripción Observaciones 
git reset _nombreArchivo.extension_
 Quita un archivo del escenario (stage) Ejemplo: 
git reset browserconfig.xml
Ejemplo: 
git reset Curso/demo-02/browserconfig.xml
 
git reset *._extension_
 Quita los archivos con extensión específica del escenario (stage) Ejemplo: 
git reset *.xml
 
git reset HEAD _nombreArchivo.extension_
 Quita los archivos del escenario (stage) y muestra un mensaje en consola Ejemplo: 
git reset HEAD README.md
Ejemplo: 
git reset HEAD Curso/demo-03/README.md
 
git reset --soft HEAD 
git reset --soft HEAD^^
El símbolo ^^ apunta al penúltimo commit y sin ^^ al último commit 		Regresa a los cambios hechos en el último commit y agrega los nuevos usando git commit -am 
git reset --mixed _hash_
Regresa en el tiempo quitando los archivos del escenario (staged) sin borrar nada físicamente Ejemplo: 
git reset --mixed 677b624
 
git reset --hard _hash_
Regresa en el tiempo borrando todo lo posterior al commit especificado Ejemplo: 
git reset --mixed 677b624
# Comando Descripción Observaciones 
git log --oneline
 Muestra los commits en una sola línea (forma resumida con hash corto) Ejemplo: 
cae0326 Creación del Curso de Git
 
git log --oneline --decorate --all --graph
 Muestra los commits de forma "elegante" en una sola línea (forma resumida con hash corto) Ejemplo: 
* cae0326 Creación del Curso de Git

git stash

                    		
                    
                      Coloca los archivos en seguimiento en un área temporal
                    
                    
                  

                  

                    
                    
                      
git stash save

                    		
                    
                      Coloca los archivos en seguimiento en un área temporal (hace lo mismo que git stash)
                    
                    
                  

                  

                    
                    
                      
git stash save _"mensajeStash"_

                    		
                    
                      Coloca los archivos en seguimiento en un área temporal (hace lo mismo que git stash)
                      y agrega un mensaje al stash
                    
                    
                      Ejemplo:
                      
git stash save "Agregamos a loki en los villanos"

                  

                  

                    
                    
                      
git stash list

                    		
                    
                      Muestra todos los stash almacenados
                    
                    
                  

                  

                    
                    
                      
git stash list --stat

                    		
                    
                      Muestra todos los stash almacenados de forma extendida
                    
                    
                  

                  

                    
                    
                      
git show stash

                    		
                    
                      Muestra información del último stash de forma extendida (es igual a poner git show stash@{0})
                    
                    
                  

                  

                    
                    
                      
git show stash@{_numStash_}

                    		
                    
                      Muestra información del último stash de forma extendida
                    
                    
                      Ejemplo:
                      
git show stash@{1}

                    		
                  

                  

                    
                    
                      
git stash pop

                    		
                    
                      Recupera el último trabajo (stash) almacenado temporalmente y borra el stash
                    
                    
                  

                  

                    
                    
                      
git stash apply

                    		
                    
                      Restaura el último registro del stash (es igual a poner git stash apply stash{0})
                    
                    
                  

                  

                    
                    
                      
git stash apply stash@{_numeroStash_}

                    		
                    
                      Restaura un stash en específico
                    
                    
                      Ejemplo:
                      
git stash apply stash@{1}

                    		
                  

                  

                    
                    
                      
git stash drop

                    		
                    
                      Borra el último trabajo (stash) almacenado temporalmente (es igual a poner git stash drop stash{0})
                    
                    
                  

                  

                    
                    
                      
git stash clear

                    		
                    
                      Borra todos los stash
                    
                    
                  

                  

                    
                    
                      
git stash save --keep-index

                    		
                    
                      Guarda todo menos los archivos que están en el stage(escenario)
                    
                    
                  

                  

                    
                    
                      
git stash save --include-untracked

                    		
                    
                      Guarda todos los archivos incluso los que no están en seguimiento
                    
                    
                  

                  

                    
                      Nota: *Se pueden tener múltiples entradas en el stash

#
              Término
              Concepto
            

            
              

                
                
                  
Hash

                		
                Código que Git proporciona para identificar commits
              

              

                
                
                  
Staged

                		
                Archivos preparados para ser agregados al commit
              

              

                
                
                  
Unstaged

                		
                Archivos no preparados para ser agregados al commit
              

              

                
                
                  
HEAD

                		
                Indica el commit actual y en qué rama(branch) se encuentra
              

              

                
                
                  
HASH corto

                		
                Funciona igual que el HASH normal solo que es más pequeño
              

              

                
                
                  
--

                		
                
                  Significa que después de -- viene una palabra o varias palabras
                  

                  Ejemplo:
                  
--oneline

                  
--oneline --all --graph

                  *Dónde cada palabra sería un comando
                		
              

              

                
                
                  
-

                		
                
                  Significa que después de - viene una letra o varias letras
                  

                  Ejemplo:
                  
-s

                  
-sae

                  *Dónde cada letra sería un comando
                		
              

              

                
                
                  
WIP

                		
                
                  Significa: Work In Progress
                  

                  Se muestra al hacer Git Stash
                		
              

            
          

        

        

        
        

          

            
              
#
              Nomenclatura
              Descripción
              Observaciones
            

            
              

                
                
                  
                    El archivo .gitignore se crea desde el editor de código 
                  

                    Click derecho, archivo nuevo o cualquier forma donde se pueda crear un nuevo archivo (desde el editor),
                    

                    Nota: Debe estar en la carpeta raíz (al mismo nivel que la carpeta oculta .git)
                  
                		
              

              

                
                  Formas de agregar archivos y carpetas a .gitignore
                
              

              

                
                
                  
_nombreArchivo.extension_

                		
                
                  Ignora el archivo especificado
                
                
                  Ejemplo:
                  
data.log

                  Ejemplo:
                  
Curso/demo-04/data.log

                		
              

              

                
                
                  
*._extension_

                		
                
                  Ignora todos los archivos con extensión específica
                
                
                  Ejemplo:
                  
*.log

                		
              

              

                
                
                  
_nombreCarpeta/_

                		
                
                  Ignora la carpeta especificada y sus archivos
                
                
                  Ejemplo:
                  
node_modules

                  Ejemplo:
                  
Curso/demo-4/node_modules
